# 夜间自动迭代 Loop 指令（footprints）

> 用法：`/loop 15m 读 LOOP-PROMPT.md，执行一轮（One iteration only）`
> 每次 loop 触发都是一段新的上下文。你的状态来自**文件 + git**，不靠记忆。
> 目标：趁用户睡觉，把这个静态站**安全地**往前推一点，早上他用 `git log` 就能逐条看到改了啥、测了没。

---

## ⭐ 北极星目标（这个 loop 到底要干嘛）

**把这个站推到大厂级、能让人「惊艳」的产品质感。** 内容里的**事实**不变，但呈现可以全面打磨：性能、移动端、健壮性、无障碍、代码质量，以及**视觉 / 交互 / 文案**。**适度、克制，别太过**（惊艳 ≠ 浮夸）。方向你自己权衡（收益高、风险低、不需用户拍板的优先），守住下面两条。

**两条守住（不是冻死，是别破坏已有价值）：**

- 🧭 **古海图风格守住**：`footprints.html` 那套美学——配色、纹理、火漆印章、排版、整体气质——是花大力气调出来的，**别改风格**。但这张地图**本身 bug 很多**，要**主动测、主动修**（交互 / 退层 / 跨国跳转 / 边界 / 渲染 bug 都修），外加性能 / 移动端 / 代码——**只是别动它的美学长相**。
- 📇 **个人经历守真**：`index.html` / `profile/data.js` 里的经历、项目、数据是**真的**（来自他 LinkedIn / 给过的文件），**别编造、别改事实、别瞎加条目**。但**文案语言可以润色**（按 `DESIGN.md` §10 大厂文案规范：动词开头、简洁、sentence case……），视觉呈现更可以放手提升。

---

## ⏰ 运行时间窗口（多伦多 America/Toronto）

只在 **00:30–10:30** 干活。每轮**开头先**跑 `TZ=America/Toronto date`：

- **< 00:30**：本轮**不动任何代码**，直接结束这一轮（空转，等下一次 tick）。
- **00:30 ≤ 现在 < 10:30**：正常执行一轮（见 §3）。
- **≥ 10:30**：**收工**——不做任何改动，往 `LOOP-LOG.md` 写一条「⏰ 到 10:30，停工等用户上班」，本轮结束。之后用户会手动停掉 loop。

> 撞 5h 用量上限：当轮会报错跑不动，**正常**。只要 Mac 保持唤醒 + loop 按 interval 继续排程，用量窗口重置后（约 5h）下一次 tick 会自己接上。不保证 100% 自动续，最坏就是少跑点，安全。

## 0. 这是什么项目（30 秒）

- 纯静态站，无构建：`index.html`（个人主页，英文/正式）+ `footprints.html`（古海图多层钻取地图）。
- 数据：`data/footprints.json`（所有地点/坐标/国家）。主页文案：`profile/data.js`。
- 设计规范：`DESIGN.md`（约束 index.html）、`DESIGN-MAP.md`（约束 footprints.html）——这就是“大厂风格”的硬规则，照它走，别自己拍脑袋。
- 本地预览：`npm start` → live-server `http://localhost:1999`。
- 测试：3 个浏览器 console 脚本，`tests/design.js`、`tests/design-map.js`、`tests/regression.js`，输出 ✅/❌。
- 已装 Playwright MCP，可无头加载页面、跑测试脚本、截图。

## 1. 绝对红线（永远不破）

1. **`photos/` 一个字节都不准动**——不新建、不改、不删、不移、不重命名。读元信息可以，写入绝对禁止。
2. **`data/*.geojson` 不准改**（world/states/borders/lakes 是 Natural Earth 底图几何，代码不碰它）。
3. **不准臆造旅行数据**——地点、年份、坐标只能用 `FOOTPRINTS.md` / `data/footprints.json` / 照片清单里**已有的事实**。任何需要用户拍板的事（先做哪个城市、哪年去的、选哪几张照片、版权确认）→ **跳过，记一笔到 LOOP-LOG.md，绝不猜**。
4. **🧭 古海图风格守住**：`footprints.html` 的视觉**美学**（配色 / 纹理 / 火漆印章 / 排版 / 气质）**别改**——但**主动找 bug、修 bug**、改性能 / 移动端 / 代码。改完跑 `tests/regression.js` + `design-map.js` 保绿。
5. **📇 个人经历守真**：经历 / 项目 / 数据是真的（LinkedIn / 用户给过的文件），**别编造、别改事实、别瞎加条目**。事实以 `profile/data.js` 现有内容为准；文案**语言可以润色**，视觉呈现可以提升。
6. **首页视觉 / 文案改动遵守 `DESIGN.md`**（含 §10 文案规范），改完必过 `tests/design.js`。视觉 / 交互打磨**适度，别太过**。
7. **不删用户内容**。不 `git push`，不部署，不发任何外部请求/邮件。本地提交即可。
8. **不催用户**、不在结尾写“下一步要不要…”。

## 2. 工作分支（第一轮先做）

- 不在 `main` 上动手。检查当前分支：若不是 `auto/overnight`，则 `git switch -c auto/overnight`（已存在就 `git switch auto/overnight`）。
- 所有提交只落在这个分支。早上用户在 `main` 上 `git log auto/overnight` 审查，喜欢就 merge，不喜欢直接删分支，零风险。
- 开工前确保工作区基于已有的未提交改动是干净的：若 `git status` 有用户遗留的未提交改动，先 `git stash`（记一笔），别把它们混进你的提交。

## 3. 每一轮只做一件事（原子化）

**一轮 = 选一个任务 → 最小实现 → 验证 → 绿了就提交 → 写日志 → 停。** 下一轮 tick 再做下一件。别贪多。

### 步骤

1. **读状态**：`git log --oneline -15`、`LOOP-LOG.md`（没有就建）、`TODO.md`。判断哪些已做。
2. **选任务**：从 §4 阶梯里挑**第一个还没做、且自包含（不需要用户拍板）**的任务。
3. **实现**：遵循 Karpathy 原则——外科手术式、最简、不加没要求的抽象、改动每一行都能追溯到任务本身。
4. **验证（必须真跑，不准嘴上说过了）**：
   - 端口 1999 没起就后台 `npm start`。
   - Playwright 加载 `http://localhost:1999/index.html` 和 `/footprints.html`，确认**控制台 0 报错**。
   - 跑相关测试脚本（把对应 `tests/*.js` 内容用 `browser_evaluate` 注入执行；regression 跑完读 `window._testFP`，design 类读它打印的 console / 在脚本尾部 `return R` 取结果）。确认**没有新增 ❌**。
   - 截图 light + dark 两套存到 `.playwright-mcp/`（地图改动也截一张）。
5. **判定**：全绿 → 提交；有红 → 修到绿，或 `git checkout` 回退本次改动，**绝不提交坏的**。
6. **提交**：中文 message，对齐仓库风格（看历史 commit），结尾加
   `Co-Authored-By: Claude Opus 4.8 (1M context) <noreply@anthropic.com>`
7. **写日志**：往 `LOOP-LOG.md` 追加一条（日期取 `date` 命令、本轮任务、改了哪些文件、测试结果 X/Y、截图路径、跳过了什么及原因）。
8. **停**。本轮结束。

## 4. 干什么（A 永远先做，其余你权衡）

服务北极星目标。每轮挑**一件**收益高、风险低、不需用户拍板的事。除 A 外不强制顺序——你按性价比自己排。

- **A. 测试健康（永远先做）**：跑全部 3 套测试（`design` / `design-map` / `regression`）。任何 ❌ → 修到绿。永不留坏测试，也不在红的状态上叠新东西。
- **B. 🧭 地图 bug 猎杀（`footprints.html`，守美学）**：用户说**地图 bug 多**。主动用 Playwright 点一遍——各层钻取、退层、跨国跳转、年份带、夜间切换、印章返回、面包屑、缩放边界——找 `regression.js` 没覆盖的 bug 并修。**守风格不变，只修坏的行为**；新发现的 bug 补一条进 `regression.js`。
- **C. 首页视觉 / 交互 / 文案打磨（`index.html`）**：按 `DESIGN.md` 冲大厂质感——布局、留白、层级、暗色、动效、微交互（hover/active/focus）、加载 / 过渡；文案按 §10 润色（**守 📇 事实**，只改语言）。**适度别太过**。每次一小块，过 `tests/design.js` + 截 light/dark 两图。
- **D. 📱 移动端适配（两页，重点——用户没在别的屏测过）**：响应式 CSS（守 8px 网格 + 字号阶梯）。Playwright 多视口截图验证不溢出、可点、地图能用、控件 / 菜单不互相遮挡：手机 390×844、大屏手机 430×932、平板 768 / 1024。
- **E. 性能**：首屏更快——`data/*.geojson` 共 ~5MB，考虑懒加载 / 按需加载 / 几何简化（简化后**另存新文件**，绝不改原 geojson）；图片 `loading=lazy`、字体、首屏关键 CSS。能量化就量化（加载时间 / 请求数）。
- **F. 健壮性 & 无障碍**：0 console 报错、边界情况、键盘可达、`:focus-visible`、对比度 AA/AAA、`prefers-reduced-motion` 降级。
- **G. 公开前加固**：写 `README.md`（运行 + 部署）；maplibre CDN 加 SRI（改完验证地图仍加载，破了回退）。⚠️ BGM 换 CC0 先别做（要下外部音频），记一笔提醒用户。
- **H. 代码质量（只在本轮已动的文件上）**：去重、简化、注释对齐周围风格。不顺手重构无关代码。

> 🧭 注：`footprints.html` 做 B/D/E/F/H + 修 bug，**别改它的美学长相**（C 的外观 / 文案升级只针对首页 `index.html`，地图上的地名是数据，不动）。
> 全做完、测试全绿、剩下都需用户拍板 → LOOP-LOG.md 写总结（列待决项），**停，不再调度下一轮**。别反复刷“没事可做”。
